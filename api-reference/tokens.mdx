# Token API

The Token API provides endpoints for launching tokens, managing positions, tracking performance, and automating token operations through OneClick's platform.

<Card icon="coins" title="Token Operations">
**Full token lifecycle management** through RESTful API endpoints:
- Launch new tokens on Solana
- Track launch status and progress
- Manage token positions and sales
- Monitor performance and analytics
</Card>

## Base URL

```
https://api.oneclick.com/api
```

All endpoints require authentication via Bearer token in the Authorization header.

---

## Token Launch Endpoints

### Launch Token

<Card icon="rocket" title="POST /tokens/launch">
**Deploy a new token to Solana blockchain**
</Card>

**Request Body:**
```json
{
  "name": "My Token",
  "symbol": "MTKN",
  "description": "A sample token description",
  "image": "https://example.com/token-image.png",
  "solAmount": 0.1,
  "platform": "PUMP_FUN",
  "walletId": "uuid-of-wallet",
  "twitter": "https://twitter.com/user/status/123",
  "metadata": {
    "telegram": "https://t.me/mytoken",
    "website": "https://mytoken.com"
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "launchJobId": "job_uuid",
    "tokenId": "token_uuid",
    "estimatedCompleteTime": "2024-01-15T10:30:00Z",
    "queuePosition": 3,
    "status": "PENDING"
  }
}
```

**Parameters:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Token name (max 32 characters) |
| `symbol` | string | Yes | Token symbol (max 10 characters) |
| `description` | string | No | Token description (max 1000 characters) |
| `image` | string | No | URL to token image |
| `solAmount` | number | Yes | SOL amount to spend (0.001-100) |
| `platform` | string | Yes | Launch platform: "PUMP_FUN", "RAYDIUM" |
| `walletId` | string | No | Specific wallet ID (uses primary if not provided) |
| `twitter` | string | No | Originating tweet URL |
| `metadata` | object | No | Additional token metadata |

## Launch Status & Monitoring

### Get Launch Status

<Card icon="clock" title="GET /tokens/launch/{launchJobId}/status">
**Check the status of a token launch**
</Card>

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "job_uuid",
    "status": "CREATING_TOKEN",
    "progress": 60,
    "estimatedCompleteTime": "2024-01-15T10:32:00Z",
    "logs": [
      {
        "timestamp": "2024-01-15T10:30:15Z",
        "level": "INFO",
        "message": "Metadata uploaded to IPFS"
      },
      {
        "timestamp": "2024-01-15T10:30:45Z", 
        "level": "INFO",
        "message": "Token contract deploying..."
      }
    ],
    "transactionHash": "tx_hash_when_available"
  }
}
```

**Status Values:**
- `PENDING` - Queued for processing
- `PROCESSING` - Launch in progress
- `UPLOADING_METADATA` - Uploading to IPFS
- `CREATING_TOKEN` - Deploying smart contract
- `BUYING_TOKENS` - Executing initial purchase
- `COMPLETED` - Successfully launched
- `FAILED` - Launch failed
- `CANCELLED` - User cancelled

## Selling Operations

### Sell Tokens

<Card icon="chart-line" title="POST /tokens/positions/{positionId}/sell">
**Sell tokens from a position**
</Card>

**Request Body:**
```json
{
  "sellType": "PERCENTAGE",
  "amount": 50,
  "slippageTolerance": 5.0,
  "priorityFee": 5000
}
```

**Parameters:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `sellType` | string | Yes | "PERCENTAGE", "TOKENS", "SOL", "USD" |
| `amount` | number | Yes | Amount based on sellType |
| `slippageTolerance` | number | No | Max slippage % (default: 5.0) |
| `priorityFee` | number | No | Priority fee in micro-lamports |

**Response:**
```json
{
  "success": true,
  "data": {
    "sellJobId": "sell_job_uuid",
    "estimatedReceive": {
      "sol": 0.075,
      "usd": 9.25
    },
    "estimatedSlippage": 2.3,
    "status": "PROCESSING"
  }
}
```

### Batch Sell

<Card icon="layer-group" title="POST /tokens/sell/batch">
**Sell multiple positions simultaneously**

**Request Body:**

```json
{

      "sellType": "PERCENTAGE",
      "amount": 25
    },
    {
      "positionId": "position2_uuid", 
      "sellType": "PERCENTAGE",
      "amount": 50
    }
  ],
  "slippageTolerance": 5.0

```

## Automation Endpoints

### Set Auto-Sell Rules

<Card icon="robot" title="POST /tokens/positions/{positionId}/auto-sell">
**Configure automated selling rules**
</Card>

**Request Body:**
```json
{
  "rules": [
    {
      "type": "TAKE_PROFIT",
      "trigger": {
        "priceMultiple": 2.0
      },
      "action": {
        "sellPercentage": 50
      }
    },
    {
      "type": "STOP_LOSS", 
      "trigger": {
        "priceMultiple": 0.5
      },
      "action": {
        "sellPercentage": 100
      }
    }
  ]
}
```

**Rule Types:**
- `TAKE_PROFIT` - Sell when price reaches target
- `STOP_LOSS` - Sell when price drops below threshold
- `TIME_BASED` - Sell after holding for specified duration
- `VOLUME_BASED` - Sell when volume conditions are met

## SDK Examples

### JavaScript/TypeScript

```typescript
import { OneClickAPI } from '@oneclick/sdk';

const api = new OneClickAPI({
  apiKey: 'your-api-key',
  baseURL: 'https://api.oneclick.com'
});

// Launch a token
const launch = await api.tokens.launch({
  name: 'My Token',
  symbol: 'MTKN',
  solAmount: 0.1,
  platform: 'PUMP_FUN'
});

// Monitor launch status
const status = await api.tokens.getLaunchStatus(launch.launchJobId);

// Get positions
const positions = await api.tokens.getPositions({
  status: 'active'
});

// Sell tokens
const sell = await api.tokens.sell(positionId, {
  sellType: 'PERCENTAGE',
  amount: 50
});
```

### Python

```python
from oneclick_sdk import OneClickAPI

api = OneClickAPI(
    api_key='your-api-key',
    base_url='https://api.oneclick.com'
)

# Launch token
launch = api.tokens.launch(
    name='My Token',
    symbol='MTKN',
    sol_amount=0.1,
    platform='PUMP_FUN'
)

# Get positions
positions = api.tokens.get_positions(status='active')

# Sell tokens
sell_result = api.tokens.sell(
    position_id=position_id,
    sell_type='PERCENTAGE',
    amount=50
)
```
